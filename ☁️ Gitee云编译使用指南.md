## 🚀 详细步骤（5 步搞定！）

---

### 第 1 步：启用 Gitee Go

```
1. 打开你的 Gitee 仓库页面
2. 点击左侧菜单：DevOps → Gitee Go
3. 点击 "立即开通 Gitee Go"
4. 选择 "免费版" → 确认开通

✅ 免费版：
   - 每月 2000 分钟
   - 每次最大构建时间 60 分钟
   - 完全够用！
```

---

### 第 2 步：确认配置文件

项目中已经为你准备好了配置文件：

```
.gitee/workflows/build-apk.yml
```

这个文件告诉 Gitee Go 如何构建 APK。

**文件内容已配置好，不用修改！**

---

### 第 3 步：推送代码触发构建

```
把代码推送到 Gitee：

git add .
git commit -m "添加云编译配置"
git push origin master

📌 推送后会自动触发构建！
```

---

### 第 4 步：查看构建进度

```
1. 回到 Gitee 仓库页面
2. 点击左侧：DevOps → Gitee Go
3. 看到正在运行的构建任务：
   
   ✅ 显示：🔨 构建魔法笔记 APK

4. 点击进入可以看实时日志

⏰ 预计构建时间：5-10 分钟
```

---

### 第 5 步：下载 APK！🎉

```
构建完成后：

1. 在构建详情页面
2. 点击顶部的 "Artifacts" 标签
3. 看到文件：魔法笔记-APK
4. 点击下载！

📂 下载后就是可以直接安装的 app-release.apk
```

---

## 📱 手动触发构建

**除了推送代码，也可以手动点击构建：**

```
1. Gitee 仓库 → DevOps → Gitee Go
2. 点击右上角 "新建任务"
3. 选择分支：master
4. 点击 "开始构建"

→ 就会立即开始云编译了！
```

## ❓ 常见问题

### Q: 构建失败怎么办？
**A:**
```
1. 点击构建任务看日志
2. 检查是否有语法错误
3. 可以重新触发构建
4. 或在群里/社区提问
```

### Q: 免费额度够用吗？
**A:**
```
✅ 完全够用！
- 每次构建约 5-10 分钟
- 每月 2000 分钟 = 可构建 200-400 次
- 个人开发者完全用不完
```

### Q: APK 保留多久？
**A:**
```
✅ 默认保留 30 天
需要时可以重新构建，或下载后保存到本地
```

```
✅ 可以！
- 不同分支可以同时构建
- 也可以加配置同时构建 debug/release
```

## 💡 小贴士

### 加快构建速度
```
✅ 利用缓存：Flutter 依赖会缓存，第二次构建更快
✅ 不用每次都构建：推代码才会自动构建
✅ 必要时手动触发即可
```

### 多架构 APK
```
如需生成 arm64 单独版本：
修改 .gitee/workflows/build-apk.yml 中的构建命令：

flutter build apk --release --split-per-abi

会生成 3 个 APK：
- app-arm64-v8a-release.apk 手机用
- app-armeabi-v7a-release.apk 旧手机用
- app-x86_64-release.apk 模拟器用
```
